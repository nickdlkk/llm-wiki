## 概述

卡茲克是一位在 AI 行業深耕三年的內容創作者和創業者，運營公眾號「數字生命卡茲克」，使命是「激發大家對 AI 的好奇」。

他把自己積累的 Prompts 和 Skills 開源，分為兩類：

## Skill 1：hv-analysis（橫縱分析法深度研究）

**橫縱分析法（Horizontal-Vertical Analysis）** — 卡茲克原創，融合：
- 索緒爾的歷時-共時分析
- 社會科學的縱向-橫截面研究設計
- 商學院案例研究法
- 競爭戰略分析

### 核心框架：雙軸分析

```
縱軸（歷時性）
  ↓
研究對象從誕生到現在的完整生命歷程
  ×
橫軸（共時性）
  ↓
當下時間截面 vs 競品全面對比
  ↓
橫縱交匯 → 綜合洞察 + 三個未來劇本
```

### 四步流程

**第一步：聯網信息收集**
- 子Agent 1（縱向）：起源、創始人背景、發展歷程、版本迭代、融資、危機
- 子Agent 2（橫向）：競品識別、特點、用戶口碑、行業對比
- 子Agent 3（複雜對象才需要）：創始人深度背景、社區討論

> 每個子Agent 的 prompt 都內嵌聯網指引（WebSearch + WebFetch），學術類必查 arxiv

**第二步：縱向分析**
- 起 origin：誕生的背景、創始團隊、行業環境
- 誕生節點：首次發布/成立時間
- 演進歷程：按時間順序所有關鍵節點
- 決策邏輯：為什麼選 A 而非 B？哪些早期決策「鎖定」了後來方向？
- 階段劃分：萌芽期、快速增長期、轉型期等

**第三步：橫向分析**
- 場景A（無直接競品）：分析為什麼沒有競品 + 未來競爭者方向
- 場景B（1-2個競品）：逐一深入對比
- 場景C（3+個競品）：選取 3-5 個代表性深度分析

**第四步：橫縱交匯洞察**
- 歷史如何塑造了當下的競爭位置
- 優勢/劣勢的歷史根源
- 未來三個劇本：最可能的、最危險的、最樂觀的

### 寫作風格借鑑

從「冷冰冰的諮詢報告」到「讓人從頭讀到尾的深度研究」：
- 節奏感：句子時長時短，段落跳躍自然
- 叙事驱动：不是羅列，是故事弧線
- 敢下判斷：每個觀點有事實支撐，用「我覺得」姿態
- 層層剝開：不直接講結論，讓讀者參與思考過程

### 輸出

- Markdown 研究報告 → PDF（自帶 `scripts/md_to_pdf.py`，基於 WeasyPrint）
- 全文總計：10,000-30,000 字

### 質檢清單（14條）

- [ ] 縱軸是叙事故事體？有因果邏輯？
- [ ] 創始人背景足夠深度？
- [ ] 每個關鍵節點都展開了？
- [ ] 決策邏輯有還原？
- [ ] 橫軸的競品場景判斷正確？
- [ ] 用戶口碑引用了真實用戶聲音？
- [ ] 橫縱交匯產出了新判斷？
- [ ] 三個劇本都有邏輯支撐？
- [ ] 寫作風格有節奏感？
- [ ] 沒有觸犯絕對禁區？
- [ ] 所有關鍵事實標注了來源？
- [ ] 搜不到的資訊誠實標注了「暫缺」？
- [ ] PDF 排版美觀？
- [ ] 總字數在範圍內？

## Skill 2：khazix-writer（公眾號長文寫作）

**核心口號**：「有見識的普通人在認真聊一件打動他的事。」

### 核心價值觀

- **永遠對世界保持好奇**：面對新工具新技術問「我能用它玩點什麼有意思的？」
- **講人話，像個活人**：大膽使用「我覺得」「我認為」，擁抱不完美
- **真誠是唯一的捷徑**：可以不寫，但絕不騙人
- **有所為有所部位**：不追逐違背價值觀的流量

### AI的角色邊界

**AI擅長做的**（放心交給AI）：
- 找證據和佐證
- 找類比和比喻
- 按確定角度擴寫
- 補充學科背景知識
- 梳理邏輯和結構建議

**必須人來做的**（AI做了會暴露）：
- 第一手觀察和真實經歷
- 核心創意角度
- 情緒的真實表達
- 數據到人物的同理心轉換

### 寫作協作流程

```
人：提供素材 + 核心觀點 + 個人經歷 + 情緒節點
  ↓
AI：補充背景知識 + 找證據類比 + 結構建議 + 按角度擴寫
  ↓
人：二次改寫（加入自己的聲音、打破節奏、補充真實細節）
  ↓
AI：按四層自檢體系檢查 → 輸出修改建議
  ↓
人：終審和定稿
```

### 五種文章原型

1. **調查實驗型**：親自下場做一件事，邊做邊報道發現
2. **產品體驗型**：拿到產品實際使用，帶著讀者一起體驗
3. **現象解讀型**：觀察到現象，層層深入分析
4. **工具分享型**：分享實用工具，用個人故事包裹
5. **方法論分享型**：系統性分享經驗，每節要有可執行行動建議

### 四層自檢體系

借鑒了軟體工程的測試金字塔：

**L1 硬性規則檢查**
- 禁用詞：「說白了」「意味著什麼」「本質上」「換句話說」等
- 禁用標點：冒號、破折號、雙引號
- 結構套話掃描

**L2 風格一致性檢查**
- 開頭：從具體事件切入？
- 節奏：長短句交替？有斷裂效果？
- 口語化：至少 8-10 個口語化表達？

**L3 內容質量檢查**
- 每個核心觀點有具體支撐？
- 知識「聊著聊著順手掏出來」？
- 文化升維自然？

**L4 活人感終審**
- 「讀完感覺是一個有見識的普通人在認真跟我聊一件事，還是AI在給我輸出資訊？」

### 絕對禁區

- 禁用套話：「首先...其次...最後」「綜上所述」「值得注意的是」
- 禁用標點：冒號 `：`、破折號 `——`、雙引號 `""`
- 高頻踩雷詞：「說白了」「意味著什麼？」「本質上」「換句話說」「不可否認」
- 假設性例子：不能編造場景，用「今天正在搞的」等真實正在發生的細節

### 寫作風格核心元素

- **節奏感**：像跟朋友聊天，不像寫報告
- **句式斷裂**：製造停頓和重量感（「黑暗森林。」「時間。流逝的本身。」）
- **層層剝開**：現象→表面解釋→更深追問→核心洞察
- **文化升維**：從具體事件連接到更大的文化/哲學/歷史參照
- **回環呼應**：前面埋的鉤子後面 callback 回來
- **謙遜鋪墊**：在觀點前先自謙，降低讀者防禦心
- **逐一展示法**：按「升番」邏輯排列（最弱的放前面，最炸的留最後）

## 對 LLM Wiki 項目的啟發

### 1. 子Agent並行研究模式（hv-analysis）

hv-analysis 用了三個子Agent並行收集縱向和橫向信息，這對 LLM Wiki 的 research workflow 有直接參考價值。可以借鑒：
- 設計 LLM Wiki 的 research workflow 時，考慮並行資訊收集
- 每個子Agent 的 prompt 嵌入聯網指引 + 明確的資訊來源優先級

### 2. 四層自檢體系（khazix-writer）

軟體工程測試金字塔啟發的四層質檢：
- L1 = 語法檢查（硬性規則）
- L2 = 單元測試（模式匹配）
- L3 = 集成測試（深度內容）
- L4 = 端到端（活人感）

這個分層框架很值得 LLM Wiki 借鑒——我們的產出（wiki實體頁）也可以設計類似的四層質檢流程。

### 3. AI/人邊界設計（khazix-writer）

明確區分「AI擅長做的」vs「必須人做的」——這是 skill 設計的核心原則。
LLM Wiki 的實體頁生成也可以設計這樣的邊界：哪些交給 AI 自動生成，哪些必須人來把控。

### 4. 寫作風格內化（hv-analysis + khazix-writer）

hv-analysis 的寫作風格章節（借鑑 khazix-writer 的風格）說明：
- 一個複雜流程 skill（hv-analysis）可以內嵌另一個 skill 的寫作風格（khazix-writer）
- 技能可以組合使用

### 5. 工具鏈整合

hv-analysis 自帶 `scripts/md_to_pdf.py`，khazix-writer 自帶 `references/` 目錄——skill 不是只有 prompt，還有配套腳本和參考資料。LLM Wiki 的實體頁生成也可以考慮配套轉換腳本（如 md → feishu docx）。
