# 张小珺对罗福莉的3.5小时访谈：AI范式已然巨变

**来源**：张小珺商业访谈录，2026-04-24，时长约3.5小时
**主持人**：张小珺（财经媒体人）

## 一、3:1:1 算力分配公式

> "For研究、For Pre-train和For Post-train，我自己觉得一个非常合理的卡的比例可能是3:1:1。Pre-train和Post-train投入的算力应该是相当的，而研究的比例应该至少是你正式起训练卡总量的还要多一点——你要额外留更多的卡来做研究。"

旧时代（GPT-4时代）：算力分配 3:1 或 5:1，全砸预训练。
新时代：研究 : 预训练 : 后训练 = 3 : 1 : 1。
预训练和后训练的算力相当——这是行业拐点信号。

**背景**：MoE + RL 已经训不动了。要追求"Loss Spike 是零"，你得在起训前用海量小规模实验把每个潜在不稳定点都摸过一遍。GRPO在Dense上一招鲜，到MoE时代已经不够了。

---

## 二、Agent入场圈 = Opus级

> "你要做到接近 Claude 4.6 Opus 的水平，那才是一个入场圈。"

想做Agent应用，基础模型能力必须接近 Opus 级。否则 Agent 跑两步就出错，业务无法成立。
国内大多数模型还没拿到这张入场券。

---

## 三、长上下文稳定性：宣传 ≠ 可用

> "现在 Claude 在长上下文能力的稳定度上，基本上只有 Claude 4.6 Opus 和 Sonnet 是理想的。其他像 Gemini，虽然宣称长上下文的能力很好，但其实都是不行的。"

两个概念区分：
- **宣传的 context 长度**：模型架构上能吃多长的输入
- **实际的 context 稳定性**：Agent业务里长输入能否稳定输出正确结果

Gemini 宣传 1M/2M context window，但实战稳定性不如 Claude。选模型时不要看模型卡片上的数字，要做实际任务的稳定性压测。

---

## 四、长任务训练的物理瓶颈

> "做长程任务的时候，其实真实的训练不会在这么长程的任务上去训。"

直接用长任务训 RL 是走不通的——单次往返 1-2 分钟，一个 batch 一天训不了几轮。
业界实际策略：
1. 预训练阶段用短任务，让模型学会单步推理、工具使用、context处理
2. 后训练阶段做长任务激活：把短任务里学到的能力组合激活到长任务上

## 五、OpenClaw 认知转变（罗福莉自述三天经历）

### 第一天：凌晨两点到六点
- 装了 OpenClaw，用到天亮，"多巴胺/内啡肽持续分泌"
- 感受："非常有自主性、有灵魂"——它会提醒你睡觉
- 原因：System MD 在每轮 Context 前拼上当前时间，是"精心编排的 Context"

### 第二天：生活和工作任务
- 把"怎么激发团队好奇心"、"怎么筛选有好奇心的人"交给它
- 它深入探讨了一小时，形成体系化的 Skills，变成了她的"数字分身"
- 用它来筛选人才、构建团队组织架构

### 第三天：研究任务
- 最关键的：在 Agent 框架里做"多轮交互"的 User Agent 模拟
- 两小时做出了一个很好的 User Agent——用来生成 Agent 场景数据做 SFT/RL

## 六、OpenClaw vs Claude Code

| | Claude Code | OpenClaw |
|---|---|---|
| 设计目标 | For 软件工程——更好写代码 | For 端到端完成所有任务 |
| 源码 | 黑盒，改不了 | 开源可改，可改 Memory 系统、Agent 架构 |
| 记忆系统 | Session级，压缩记忆 | 分层分级，更持久 |
| 多模型协作 | 需要自己配 | 自动调度最好的模型 |
| 适用场景 | 严肃编程 | 非严肃编程、日常生活、研究 |

**核心洞见**："一个非常复杂的 Agent 框架设计，能够弥补非常多模型能力的短板。"

**Opus 的角色**：先用 Opus 4.6 把框架改好，再用 Sonnet 或国内模型接进去，也很强大。框架 + 模型要共同进步。

## 七、Agent 框架的意义

- Agent 框架是人和模型之间的"厚中间层"——定义交互层、定义怎么跟模型沟通、知道模型能力长板短板、怎么做调度
- 前端 UI 是最薄的一层，已经不关键了
- 开源 Agent 框架激发社区创造力——看到别人用它做成某件事，会激发自己的想象力
- 想象力是沉积的

## 十三、生产力变革时代（罗福莉访谈原文）

> **罗福莉**：26年的主要旋律是生产力变革，高生产力场景的持续突破。我们要做更长程的任务，强调多Agent之间的协作（更复杂任务不可能是单一Agent做的）。多Agent协作某种程度上也是为了成本和时间的考量。以及怎么激发Agent的群体智慧。现在市场上已有的Multi-Agent工作有点伪——说依赖Multi-Agent能实现更好的最终任务完成率，这个维度上有点伪。但它能提升效率（任务完成的速度）和节省成本，这是确定的。我没有看到Multi-Agent一定能够实现更高上限的东西，但会发展，只是目前还没看到。

> **罗福莉**：就是一个生产力加速变革的时代。很多人会觉得很多工作不需要自己做了。这时候人更应该思考自己的意义和价值是什么。Agent能做什么样的高价值任务变得更重要了。更高价值的任务意味着更长的Context、更多的Token消耗量，意味着能替代最顶尖那一群人的智能就够了。下边那群人的智能可以慢慢以另外的方式替代掉（比如Robotics）。

> **罗福莉**：如果要For更强的某些能力，完全For更高价值的场景做是更有价值的。另一个演变角度是：做一个对全社会更有益、对所有人都有帮助的模型，就不应该只For更高价值场景，而是让所有人都能感受到模型的智能水平。那可能是另一种做法：更注重多模态（尤其是对视频的理解、对细微环境的理解），注重成本——成本是产生革命非常重要的因素。你不能完成一个任务要花1000刀，必须要达到一个很高的替换系数比，能帮我节省十倍的成本我才会考虑去试。所以你怎么做一个更低成本、更高效率、更高速度的Agent框架以及对应的模型，就很关键。
